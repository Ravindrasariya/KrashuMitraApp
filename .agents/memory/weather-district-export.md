# Open-Meteo district weather export

Script `script/historical-weather-export-districts.mjs` pulls Open-Meteo archive data for 63
districts on the **free tier only** (no API key on the archive endpoint) and writes one big xlsx.

## Free-tier 429 cycling
- The archive endpoint rate-limits hard. The script fetches one city at a time, caches each as
  `exports/_cache/<name>.json` (full city only written when all hourly chunks + daily complete),
  and throws a `RateLimitStop` sentinel on HTTP 429 so a re-run resumes from cache.
- To pull everything you must cycle: probe the endpoint until it returns 200, restart the workflow,
  let it grab 1–3 cities, repeat. Quota windows reopen anywhere from ~5 to ~20+ min apart.
- **Why:** there is no key — the only way to finish is patience across many quota windows.

## Cache trailing-row anomaly
- Some caches generated by *sibling* export scripts contain one extra trailing day (one day past
  this script's `END_DATE`, e.g. a `2026-05-17` row when END_DATE is `2026-05-16`). This breaks the
  script's uniform-grid check (`<city> has 5981 rows, expected 5980`) at workbook-build time.
- **Fix:** trim each offending cache to the `[START_DATE, END_DATE]` range (and dedupe by date).
  Affected the default metro cities (Indore, Hyderabad, Surat, Mumbai, Patna) in one run.
- **How to apply:** if the build fails the uniform-grid check, filter rows by date range in the
  `exports/_cache/*.json` files before rebuilding — do not re-fetch.

## SheetJS write is pathologically slow
- `XLSX.writeFile` over the full grid (63 cities × 5980 rows × 28 cols ≈ 377k rows) is single-threaded
  CPU-bound and takes **tens of minutes of CPU** (observed 30–40+ min at 100% CPU), with RSS flat
  around ~1.9 GB. It is NOT hung — confirm via rising utime+stime in `/proc/<pid>/stat`.
- A prior all-cities run produced an 82 MB xlsx, so the pipeline does complete; just let it run.
- **How to apply:** run the build via the workflow (no bash timeout) and poll the filesystem for the
  output file; don't kill it assuming a hang.
