How It Works

Two environment secrets are used:

    RECAPTCHA_SECRET_KEY — backend server key (for verifying tokens with Google)
    VITE_RECAPTCHA_SITE_KEY — frontend site key (for rendering the CAPTCHA widget)

Frontend (Login.tsx)

    Uses the react-google-recaptcha npm package to render Google reCAPTCHA v2 widget
    Dev skip logic (line 43):

    const RECAPTCHA_SITE_KEY = import.meta.env.DEV ? "" : (import.meta.env.VITE_RECAPTCHA_SITE_KEY || "");

    In development (import.meta.env.DEV is true), the site key is set to empty string "".
    The CAPTCHA widget only renders when RECAPTCHA_SITE_KEY is truthy (line 271): {RECAPTCHA_SITE_KEY && (...)} — so in dev, the widget never shows.
    The login button is only disabled by CAPTCHA when the site key exists (line 286): disabled={isLoading || (RECAPTCHA_SITE_KEY && !captchaToken)}
    The "please complete CAPTCHA" validation (lines 75, 160) is also gated by RECAPTCHA_SITE_KEY being truthy.

Result in dev: No CAPTCHA widget, no token required, login works freely.
Backend (routes.ts)

    verifyCaptcha helper (lines 8-27): Sends the token to Google's siteverify endpoint. If RECAPTCHA_SECRET_KEY is not set, it returns true (skips verification).
    Login route (lines 3220-3230):

    const isDevelopment = process.env.NODE_ENV === 'development';
    if (process.env.RECAPTCHA_SECRET_KEY && !isDevelopment) {
      // Only enforce CAPTCHA in production with key configured
    }

    CAPTCHA is enforced only when both conditions are true: the secret key exists AND it's not development mode.

Result in dev: Even if the secret key is somehow set, the isDevelopment check skips enforcement.
Summary
Layer	Dev Mode	Production
Frontend	Site key forced to "" → widget hidden, no token needed	Widget shown, token required before login
Backend	isDevelopment flag skips enforcement	Token verified with Google if secret key exists

It's a double-gated approach — both frontend and backend independently skip CAPTCHA in development, so login works without any friction during development.
